# 서브에이전트 번역 운영 규칙

대규모 문서(800줄+, div/HTML 포함) 번역 시 서브에이전트 실패 빈도가 높다. 이유: 긴 HTML 구조를 유지하면서 텍스트만 교체하는 작업이 LLM에게 인지 부하가 크기 때문이다.

| # | 규칙 | 이유 |
|---|------|------|
| 1 | **1언어 1에이전트** | 병렬 처리 + 실패 격리. 한 에이전트가 2개 언어 = 두 번째 품질↓ |
| 2 | **구조 템플릿 선제공** | 문서 골격을 에이전트에게 먼저 전달하고 '텍스트만 채워라'로 지시 |
| 3 | **섹션 단위 검증** | §1~§4, 부록 단위로 구조 일치 확인. 첫 §1 완료 시점에 즉시 qc_diff.py 실행 |
| 3-b | **구조 QC 선행 (pre-flight)** | 텍스트 번역 착수 전에 빈 골격(skeleton) 생성 후 qc_diff.py 검증. PASS 후에만 텍스트 채우기 |
| 4 | **실패 시 전체 재시도** | 텍스트 품질 FAIL = 전체 재작성. 단, 구조만 FAIL이면 외과적 수정 허용 |
| 5 | **정량적 완료 기준** | 헤딩·콜아웃·div 수 = 원본 100% 일치 |
| 6 | **2패스 순차 실행** | 동일 에이전트가 1패스(직역)→2패스(윤문) 순차. 1패스 건너뛰고 의역 직행 금지 |
| 7 | **역번역 별도 에이전트** | 번역 ≠ 역번역 에이전트. 자기 번역 자기 역번역 = 원본 기억 오염 |
| 8 | **채점 별도 에이전트** | 번역 ≠ 채점 에이전트. 자기 평가 편향 방지. 4축 평균 ≥4.0 + 개별 ≥3.0 |
| 9 | **언어당 호출 예산** | 풀패스: 골격검증 + 번역(2패스) + 역번역 + 채점 = 4~5회/언어. 5개 언어 = 20~25회 |

**수정4 적용:** 번역 산출물 편차 수정 시 trigger-dictionary §6 수정4 프로토콜 적용 — 레벨 판정→게이트→외과적 실행→POST_VERIFY(전파맵). grep 잔존 검사 필수.

## 언어별 특성

| 언어 | 격식체 | 길이 변동 | 주의 |
|------|--------|----------|------|
| **EN** | 서술형 격식체 | ±0% (기준) | 도메인 표준 용어 매칭 (IB/Oxford) |
| **CN (简体)** | 간체 공식 표기 | -20~30% | 짧아서 div 여백 불균형 가능. 간체/번체 혼용 금지 |
| **JP** | 敬体(です・ます) | +10~20% | 경어·조사로 길어짐. だ체 혼입 주의 |
| **TH** | ภาษาทางการ | 바이트 +40% | UTF-8 멀티바이트. 서브에이전트 실패율 高 |
| **ID** | EYD V 공식 표기 | ±5% | 콜아웃·div 구조 편차 빈도 1순위. 영어 차용어 범위 넓음 |
| **KO** | 합쇼체/해요체 | 원본 | 제출청소 시 합쇼체 통일 |

## Gotchas (서브에이전트)

- **가장 흔한 실패**: 번역 내용은 맞는데 구조가 원본과 다름. apple-design-style 문서의 div border/padding 불일치도 빈번.
- **ID 특별 주의**: 5개 언어 중 구조 편차 1순위. 콜아웃 누락 + div 변경 + 신규 섹션 동시 발생.
- **TH 파일 크기**: +40%는 정상이지만 매번 원인 확인 필요.
- **TOC 동기화**: 헤딩 현지어 변경 시 TOC 위키링크도 같이.
- **역방향 수정 유혹**: 번역본 오류 발견 시 번역본만 고치면 원본 불일치. 반드시 원본→번역본.
- **부분 패치 함정**: 80% 완성 + 수동 패치 = 톤·격식체 달라짐. 차라리 전체 재시도.
- **골격 검증 스킵**: "간단한 문서니까" 골격 생략 = 텍스트 완성 후 누락 발견 → 전체 재작업.
- **수정4 미적용**: 직접 패치 = old 잔존(잡탕)·중복삽입.
- **qc_diff.py 미실행/FAIL 무시**: 눈으로 "비슷"하다고 넘기면 div style 1px 차이·TOC 깨짐 놓침.
- **의도 주석 생략**: "간단한 문서니까" 생략 = 톤 뒤섞임.
- **2패스 역할 혼합**: 1패스 자연스러움↑ = 정확성↓, 2패스 의미 변경 = 충실성↓.
- **채점 = 번역 동일 에이전트**: 자기 채점 관대해짐.
- **네이티브 표현 사전 미갱신**: 새 도메인 표현 미갱신 시 다음 번역에서 또 직역.
