# 다국어 번역 워크플로우 상세

## 번역 실행 워크플로우 전체

```
원본(SoT) 확인 → 의도 주석 작성 → 번역 순서 결정 → 서브에이전트 배정
→ 1패스(충실 직역) → 2패스(네이티브 윤문) → 백트랜슬레이션 검증
→ 채점 루브릭 평가 → 구조 QC(qc_diff.py) → 편차 수정
```

## 번역 순서 원칙

원본(SoT) → 영어(EN) → 나머지 언어 병렬.

**실제 병렬화:** EN 완료 후 나머지 4언어(CN·JP·TH·ID)는 병렬 실행. '1언어 1에이전트' 원칙 유지하되 순차 대기 금지. EN→나머지 순차는 EN이 구조 기준이 되기 때문이지, 나머지 간 순차가 필요해서가 아님.

역방향 수정(번역본→원본)은 금지 — 반드시 원본을 먼저 갱신 후 파생물 동기.

## 공통 번역 원칙

| # | 원칙 | 이유 |
|---|------|------|
| 1 | **파일명은 영어** | 비해당어 사용자 핸들링. `_EN.md`, `_CN.md` 등 접미사 |
| 2 | **고유명사 보존** | 브랜드명·기관명·인명은 번역하지 않음. 현지어 설명은 괄호 병기만 |
| 3 | **주요 카피 영어 보존** | 프로그램명·섹션 타이틀·슬로건 등 브랜드 카피는 영어 원문 유지 |
| 4 | **금액 병기** | 원본 통화 + 괄호 현지 통화. 예: `400만원 (약 2.1만 RMB)` |
| 5 | **현지 맥락 괄호 추가** | 원본의 로컬 맥락을 번역 + 현지 대응 병기. 예: `韩国妈妈论坛（对应小红书）` |
| 6 | **톤 통일** | 격식체 기준 명시 후 전문 일관 유지. 번역자 재량에 의한 톤 변경 금지 |

## 버전 표기 규칙

| 항목 | 규칙 | 예시 |
|------|------|------|
| 파일명 | `{원본파일명}_{LANG}.md` | `프로그램_v1.0_EN.md` |
| 제목 | 원본 제목 + `v{M}.{m} {LANG}` | `Program v1.0 EN` |
| 부제 | `Translated from {원본언어} original v{M}.{m}` | `Translated from Korean original v1.0` |
| 헤딩 | 직역 아닌 해당 언어 고유 표현으로 치환 | "캠프의 출발점" → EN "Starting Point" |
